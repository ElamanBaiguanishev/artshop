# Artshop

Интернет-магазин авторских работ ручной работы: живопись, брелоки, декор.
Реальный проект для художницы из Казахстана, продающей по СНГ и в Европу.

> **Статус:** проектирование завершено, идёт реализация. Ссылка на работающий сайт
> появится здесь после первого релиза.

---

## Что это

Не витрина-шаблон и не учебный магазин. Живой продукт с конкретными ограничениями,
которые определили архитектуру:

- **работы существуют в единственном экземпляре** - значит остаток равен единице,
  нужен резерв при заявке и корректная обработка гонки за один товар;
- **покупка оформляется как заказ с согласованием**, а не чекаут с корзиной -
  стоимость доставки хрупкого негабарита в другую страну нельзя посчитать заранее;
- **трафик приходит с телефона** из Instagram и Pinterest, часто на медленной сети -
  отсюда SSR, агрессивная работа с изображениями и мобильная вёрстка в первую очередь;
- **отправка за границу** - мультивалютность, таможенные данные, разные товарные
  позиции у оригинальной живописи и у сувениров;
- **админкой пользуется не разработчик** - критерий приёмки: владелица ведёт каталог
  с телефона и ни разу не обращается за помощью.

## Стек

| Слой | Технология |
|---|---|
| Фронт | Next.js (App Router), TypeScript, Tailwind, shadcn/ui |
| API | NestJS, TypeScript |
| Данные | PostgreSQL, Drizzle ORM |
| Очереди | BullMQ, Redis |
| Хранилище | S3-совместимое |
| Инфраструктура | Docker, Dokploy |

Три развёртываемые части: `web`, `api`, `worker`. Разделение проведено по способу
выполнения - рендеринг, синхронные операции и фоновая работа живут по разным правилам.
Дробить магазин такого размера сильнее было бы оверинжинирингом.

## Ключевые технические решения

Подробные обоснования - в [docs/architecture.md](docs/architecture.md).

**Outbox для всего, что уходит наружу.** Уведомления в Telegram, события в Conversions
API, письма пишутся в таблицу в одной транзакции с бизнес-изменением, отправляет воркер.
Заявка не должна теряться из-за недоступности внешнего сервиса, а ответ покупателю
не должен ждать сторонних API.

**Резерв уникального товара условным UPDATE.** Перевод в `reserved` выполняется
запросом с условием на текущий статус и проверкой числа затронутых строк. Условие
внутри UPDATE уже атомарно - распределённая блокировка здесь была бы лишней сущностью.

**Дедупликация событий аналитики.** Браузерный пиксель режется блокировщиками, поэтому
те же события дублируются с сервера. Оба потока несут общий `event_id`, по которому
площадка склеивает их в одно событие.

**Деньги целыми числами в минорных единицах**, курс валюты фиксируется в момент
согласования заказа. Иначе названная в переписке сумма расходится с суммой в системе.

**Асинхронная обработка изображений.** Оригиналы загружаются напрямую в хранилище
по presigned URL, минуя API; воркер генерирует варианты, WebP и AVIF, водяной знак
и заглушку для прогрессивной загрузки.
